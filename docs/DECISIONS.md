# Project Decisions

## Framework

Decision: Use Ruby on Rails.

Reason: Rails fits the project well because Forge is primarily a database-backed CRUD app. I also have prior Rails experience from bootcamp, and using Rails makes the project feel energizing enough to continue.

## Database

Decision: Use PostgreSQL.

Reason: PostgreSQL is a strong default relational database and will support structured workout history, notes, future analytics, and possible AI features.

## Styling

Decision: Start with custom CSS instead of Tailwind.

Reason: I am comfortable writing custom CSS, and avoiding Tailwind keeps the first version simpler. Tailwind can be added later if desired.

## AI

Decision: Do not add AI in Version 1.

Reason: The first goal is to replace Apple Notes with a useful workout logger. AI should be earned after the basic app is useful.

## Deployment

Decision: Develop locally first.

Reason: Local development is enough until the app proves useful. Synology NAS deployment is a future option, but not needed for the first milestone.

## Product Scope

Decision: Build for personal use first.

Reason: The best first customer is me. Commercial viability can be considered later, but the app should first solve my own real workflow.
